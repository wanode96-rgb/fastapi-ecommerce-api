# --- WHAT IS A CONTEXT MANAGER? ---
# 1. Definition: A Python tool that manages a resource (File, DB, Socket).
# 2. Duty: Guaranteed "Setup" (Open) and "Cleanup" (Close).
# 3. Security: Prevents memory leaks and locked database connections.

# --- with (Sync Context Manager) ---
# Usage: Local tasks (Files).
# Why: Stops everything until the task is done (Blocking).

with open("data.txt", "r") as f:
    # __enter__() opens file here
    print(f.read())
# __exit__() closes file here (Even if an error occurs!)

# --- async with (Async Context Manager) ---
# Usage: Network tasks (FastAPI, PostgreSQL).
# Why: Lets other users access the API while waiting (Non-Blocking).

async with session_factory() as session:
    # __aenter__() connects to DB here
    await session.execute(query)
# __aexit__() disconnects from DB here automatically


Python Configuration Classes (The Config Sub-class)

In libraries like Pydantic and SQLModel, the class Config is a special "internal" sub-class used to change how the main class behaves.

1.  class Config in Settings

  - Purpose: Tells Pydantic how to load your secret variables (API keys, Database URLs).
  - Key Setting: env_file = ".env" (Automates loading from the .env file).

2.  class Config in Schemas

  - Purpose: Tells Pydantic how to read data from a Database Object instead of a Dictionary.
  - Key Setting: from_attributes = True (Allows converting DB Objects to JSON).

Comparison:

  - Settings Config: Handles INPUT (loading secrets from .env).
  - Schema Config: Handles OUTPUT (formatting DB data into JSON for the API).
